Prototype-based Programming

What is it?

Prototype-based programming

“Prototype-based programming is a style of object-oriented programming in which behaviour reuse (known as inheritance) is performed via a process of cloning existing objects that serve as prototypes.

This model can also be known as prototypal, prototype-oriented, classless, or instance-based programming. Delegation is the language feature that supports prototype-based programming.”

WHAT

is prototype in JS?

Prototype Object

“JavaScript objects are dynamic "bags" of properties (referred to as own properties).

JavaScript objects have a link to a prototype object.

When trying to access a property of an object, the property will not only be sought on the object but on the prototype of the object, the prototype of the prototype, and so on until either a property with a matching name is found or the end of the prototype chain is reached.”

Prototype Chain

All about objects being linked to other objects

Prototype property

  • Link of an object to its prototype object
  • Is not enumerable ( not accessible in for/in loop )
  • __proto__ "pseudo" property ( browsers )

Code


var user = {
	"name": "prototype",
	"getName": function() {
		return this.name;
	}
};

console.log( "Name: %s", user.name );
// Is there a 'name' property on user? Yes
// Name: prototype

console.log( "Name: %s", user.getName() );
// Is there a 'getName' method on user? Yes
// Name: prototype

console.log( "Has name property: %s", user.hasOwnProperty( "name" ) );
// Is there a hasOwnProperty method on user? No, check its prototype
// Is there a hasOwnProperty method on the prototype of user ? Yes
// Has name property: true

console.log( Object.getPrototypeOf( user ) );
// Object {
//	...
//	hasOwnProperty: function hasOwnProperty() { ... },
//	...
// }

Inheritance

Prototype and Inheritance

“JavaScript is a bit confusing for developers experienced in class-based languages (like Java or C++), as it is dynamic and does not provide a class implementation per se (the class keyword is introduced in ES6, but is syntactical sugar, JavaScript remaining prototype-based).”
“When it comes to inheritance, JavaScript only has one construct: objects. Each object has an internal link to another object called its prototype. That prototype object has a prototype of its own, and so on until an object is reached with null as its prototype. null, by definition, has no prototype, and acts as the final link in this prototype chain.”

Prototype and Inheritance

  • Create a function (constructor)
  • Add methods and properties on the function's prototype property

Instances of that function will have those methods and properties.

Constructor Property

All objects that inherit from another object inherit a constructor property.

It is a property that points to the constructor of the object.

Prototype and Inheritance (ES5)


/**
 * Class: User
 */
function User( name ) {

	this.name = name;

}

User.prototype.getName = function() {

	return this.name;

};


/**
 * Class: Admin
 */
function Admin( name ) {

	User.call( this, name );

}

// Create an Admin.prototype object that inherits from User.prototype
// Object.create() creates a new object with the specified prototype object and properties
// Object.create( proto[, propertiesObject] )
Admin.prototype = Object.create( User.prototype );

// Set the "constructor" property to refer to Admin
Admin.prototype.constructor = Admin;


var admin1 = new Admin( "hello" );

console.log( "Is admin1 an instanceof Admin? %s", admin1 instanceof Admin );
// Is admin1 an instanceof Admin? true

console.log( "Is admin1 an instanceof User? %s", admin1 instanceof User );
// Is admin1 an instanceof User? true

console.log( "Name: %s", admin1.getName() );
// Name: hello

Prototype and Inheritance (ES6)


class User {

	constructor( name ) {
		this.name = name;
	}

	getName() {
		return this.name;
	}

}

class Admin extends User {

	constructor( name ) {
		super( name );
	}

	getName() {
		return ( "admin: " + super.getName() );
	}

}

Design Patterns

Constructor

“By simply prefixing a call to a constructor function with the keyword new, we can tell JavaScript we would like the function to behave like a constructor and instantiate a new object with the members defined by that function.”

Constructor


/**
 * Class: User
 */
function User( name ) {

	if ( this instanceof User ) {
		this.name = name;
		return this;
	} else {
		return new User( name );
	}

}

var user1 = new User( "hello" );

Factory

“A Factory can provide a generic interface for creating objects, where we can specify the type of factory object we wish to be created.”

Factory


function ComponentFactory( type ) {

	var component = null;

	if ( type === "C" ) {
		component = new Comp();
	} else if ( type === "S" ) {
		component = new Symbol();
	}

	return component;

}

var comp1 = ComponentFactory( "C" );
var symbol1 = ComponentFactory( "S" );

Module

“In JavaScript, the Module pattern is used to further emulate the concept of classes in such a way that we’re able to include both public/private methods and variables inside a single object, thus shielding particular parts from the global scope.”

Module


var module = (function() {

	// private variable
	var value = "private value";

	return {
		// public method
		"getValue": function() {
			return value;
		}
	};

})();

console.log( module.getValue() );
// private value

Singleton

“There must be exactly one instance of a class, and it must be accessible to clients from a well-known access point.”

Singleton


var Singleton = (function () {

	// private
    var instance;

    function createInstance() {

		var object = {
			"name": ""
		};

        return object;
    }

    return {
        getInstance: function () {

            if ( !instance ) {
                instance = createInstance();
            }

            return instance;
        }
    };
})();

var instance1 = Singleton.getInstance();
var instance2 = Singleton.getInstance();
console.log( 'Same instance? ', instance1 === instance2 );
